Создание бота и получение токена

⏱️ ~5 мин

Первым делом нам нужно зарегистрировать бота в Telegram через BotFather.

Инструкция:

  1. Откройте Telegram и найдите бота @BotFather.
  2. Нажмите Start и отправьте команду /newbot.
  3. Введите имя бота (например: HR Assistant Demo).
  4. Введите юзернейм (должен заканчиваться на bot, например: my_hr_demo_bot).
  5. BotFather пришлет HTTP API Token — длинную строку вида:
    560875432:AAHf8G... (Скопируйте это!)
📥 Куда вставлять в n8n:
  1. В n8n перейдите в меню слева: Credentials.
  2. Нажмите кнопку Add Credential.
  3. В поиске напишите Telegram API и выберите его.
  4. Вставьте скопированный токен в поле Access Token.
  5. Назовите его Telegram Bot и нажмите Save.
💡
Совет: Сразу отправьте боту /start в Telegram, чтобы активировать чат. Без этого бот не сможет отправлять вам сообщения первым.

API Keys

Получение ключей API

⏱️ ~15 мин

Для работы бота нужны три ключа: Pinecone (векторная база), OpenAI (эмбеддинги), OpenRouter (мозг).

1. Pinecone (Векторная база)

Free

Pinecone хранит ваши данные в виде векторов и позволяет мгновенно находить релевантные фрагменты. Бесплатный тариф покрывает до 100,000 записей.

  1. Перейдите на app.pinecone.io и зарегистрируйтесь (можно через Google).
  2. Нажмите Create Index.
  3. Заполните поля:
    • Name: legal-assistant (или любое другое)
    • Dimensions: 1536 (стандарт для OpenAI text-embedding-3-small)
    • Metric: cosine (рекомендуется для текстового поиска)
  4. Нажмите Create Index и дождитесь создания.
  5. В меню слева выберите API Keys → Скопируйте ключ.
⚠️
Важно! Запишите также Environment URL вашего индекса (он понадобится в HTTP-запросах). Его видно на странице индекса: выглядит как https://your-index-xxxxx.svc.xxxxx.pinecone.io
📥 Куда вставлять в n8n:
  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите Pinecone API.
  3. Вставьте скопированный API Key.
  4. Назовите его Pinecone и нажмите Save.

Альтернатива: В нашем воркфлоу Pinecone используется через HTTP Request (не через встроенную ноду). В этом случае API Key вставляется прямо в заголовок запроса — это уже настроено в скачанном JSON.

2. OpenAI (Для Эмбеддингов)

$5 Credit

OpenAI используется только для превращения текста в векторы (эмбеддинги). Модель text-embedding-3-small стоит копейки — $0.02 за 1 млн токенов.

  1. Перейдите на platform.openai.com/api-keys.
  2. Войдите или зарегистрируйтесь.
  3. Нажмите Create new secret key.
  4. Придумайте название (например: n8n-embeddings).
  5. Скопируйте ключ sk-....
    🔴
    Ключ показывается только один раз! Сразу скопируйте и сохраните его в надежное место.
  6. Перейдите в Settings → Billing и пополните баланс минимум на $5 (Credit Card или Debit Card).
📥 Куда вставлять в n8n:
  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите OpenAI API (или OpenAi API).
  3. Вставьте ключ sk-... в поле API Key.
  4. Назовите его OpenAI account и нажмите Save.
💡
Расходы: Для загрузки 100 документов в Pinecone уйдет примерно $0.01. Для 1000 запросов пользователей — еще $0.02. Это очень дешево!

3. OpenRouter (Мозг — DeepSeek / GPT)

от $2

OpenRouter — это единый шлюз к разным AI моделям (DeepSeek-V3, GPT-4o, Claude 3.5). Работает без VPN из РФ и принимает крипту.

💡
Рекомендуемая модель: deepseek/deepseek-chat — быстрая, умная и очень дешевая. Для RAG-бота идеальна!
Альтернатива: google/gemini-2.0-flash-001 — бесплатный лимит для старта.
💳 Как оплатить из РФ через Zarub:

Сервис Zarub позволяет выпустить зарубежную виртуальную карту для оплаты OpenRouter, OpenAI и других сервисов.

  1. Перейдите в Telegram-бот @zarub_robot.
  2. Пополнение: Минимальная сумма — $10. Можно пополнить с российской карты или криптой.
  3. Выпуск карты: В приложении нажмите "Выпустить карту". Пополните баланс кошелька. Нажмите зеленую кнопку (перевести на карту), чтобы деньги попали на счет карты.
  4. Оплата: Нажмите фиолетовую кнопку — она покажет реквизиты карты. Используйте их при привязке карты в OpenRouter.

Получение API ключа OpenRouter:

  1. Зайдите на openrouter.ai и зарегистрируйтесь.
  2. Перейдите в раздел Credits → Add credits и пополните баланс на $2+.
  3. Перейдите в раздел Keys → Create Key.
  4. Придумайте название (например: n8n-rag-bot).
  5. Скопируйте ключ sk-or-....
📥 Куда вставлять в n8n (ВНИМАТЕЛЬНО!):

OpenRouter совместим с OpenAI API, поэтому в n8n мы создаем его как OpenAI Credential с кастомным URL:

  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите OpenAI API (да, именно OpenAI, не OpenRouter!).
  3. Вставьте ключ sk-or-... в поле API Key.
  4. Раскройте Options (внизу) и в поле Base URL впишите:
    https://openrouter.ai/api/v1
  5. Назовите его OpenRouter и нажмите Save.
⚠️
Частая ошибка: Если забыть указать Base URL — n8n будет слать запросы в OpenAI вместо OpenRouter, и ключ не сработает. Обязательно проверьте, что Base URL = https://openrouter.ai/api/v1
📋 Итого — у вас должно быть 3 Credentials в n8n:
Telegram Bot Telegram API Токен от BotFather
OpenAI account OpenAI API Ключ sk-... (для эмбеддингов)
OpenRouter OpenAI API + Base URL Ключ sk-or-... (для генерации)

База Знаний

Подготовка данных

⏱️ ~3 мин

Для демо мы используем встроенные данные (Hardcoded JSON) в воркфлоу загрузки. Вам НЕ НУЖНО подключать Google Sheets прямо сейчас.

В файле Legal_RAG_Ingestion уже прописаны 7 статей законов и HR-правил (УК РФ, ТК РФ, ГК РФ).

💡
Как это работает: Данные → разбиваются на чанки (куски) → превращаются в векторы (эмбеддинги) → загружаются в Pinecone. При запросе пользователя бот ищет ближайшие по смыслу чанки и передает их LLM для генерации ответа.

Хотите свои данные?

Если хотите заменить демо-данные на свои регламенты:

  1. Откройте узел "Load Legal Data" (Code Node) в Ingestion воркфлоу.
  2. Замените текст в массиве legalData на ваши правила. Каждый элемент — это один документ с полями:
    • id — уникальный идентификатор
    • content — текст документа
    • metadata — мета-данные (тип, код, статья)
  3. Сохраните и запустите воркфлоу.

Импорт и настройка воркфлоу

⏱️ ~10 мин

Теперь импортируем оба воркфлоу в n8n и подставим ваши ключи.

1. Загрузка базы (Ingestion)

  1. В n8n нажмите + Add Workflow → Import from File.
  2. Загрузите файл Legal_RAG_Ingestion.json (или вставьте JSON).
  3. Откройте узел "OpenAI Embeddings" — выберите ваш Credential OpenAI account.
  4. Откройте узел "Pinecone Upsert" — замените YOUR_PINECONE_API_KEY на ваш реальный ключ в Headers.
  5. Также замените URL Pinecone на ваш реальный URL индекса.
  6. Нажмите Execute Workflow (кнопка "Test workflow" внизу).
  7. Следите за зелеными галочками. Если все ноды прошли успешно — данные загружены в Pinecone!
✅
Проверка: Зайдите в Pinecone Console → ваш индекс → вкладка Vectors. Должно быть 7+ записей.

2. Чат-бот (Chat)

  1. Создайте новый воркфлоу и импортируйте Legal_RAG_Chat.json.
  2. Настройте ноды (кликните на каждую и выберите нужный Credential):
    • Telegram Trigger → Credential: Telegram Bot
    • Embed Question → Credential: OpenAI account
    • Pinecone Query → замените YOUR_PINECONE_API_KEY и URL индекса
    • Legal AI Response → Credential: OpenRouter
    • Send Telegram Response → Credential: Telegram Bot
  3. Нажмите переключатель Active вверху справа (чтобы бот слушал сообщения).
⚠️
Важно: В узле Legal AI Response убедитесь, что выбрана нужная модель. По умолчанию стоит gpt-4o-mini — замените на модель из OpenRouter, например deepseek/deepseek-chat.

⏱️ ~5 мин

Финальный этап — проверяем, что все работает!

  1. Убедитесь, что Chat воркфлоу Active (зеленый переключатель).
  2. Откройте вашего бота в Telegram.
  3. Напишите: Можно ли работать с Бали?
  4. Бот должен ответить со ссылками на статьи ТК РФ.
  5. Попробуйте еще: Какое наказание за кражу? или Как расторгнуть договор аренды?
🔴
Бот не отвечает? Проверьте:
  • Workflow активирован (переключатель Active зеленый)
  • Все Credentials настроены правильно
  • Нет ошибок во вкладке Executions
  • URL Pinecone и API ключ подставлены верно
  • OpenRouter Credential имеет правильный Base URL
🎉
Поздравляем! Если бот отвечает — вы только что запустили свою RAG-систему! Теперь можно загрузить свои реальные документы и использовать бота для HR-консультаций.

❓ Частые вопросы и проблемы

Бот не реагирует на сообщения ▼

1. Убедитесь, что Chat воркфлоу активирован (зеленый переключатель Active).

2. Проверьте вкладку Executions — там видны все запуски и ошибки.

3. Откройте Telegram Trigger ноду и убедитесь, что выбран правильный Credential.

4. Попробуйте деактивировать и снова активировать воркфлоу.

Ошибка "Invalid API Key" в OpenRouter ▼

1. Проверьте, что ключ начинается с sk-or-.

2. Убедитесь, что на балансе OpenRouter есть средства (хотя бы $1).

3. Проверьте, что в Credential указан Base URL: https://openrouter.ai/api/v1

4. Попробуйте создать новый ключ и подставить его заново.

Pinecone: "Index not found" или пустые результаты ▼

1. Проверьте, что URL индекса в HTTP Request нодах совпадает с реальным URL из Pinecone Console.

2. Убедитесь, что Ingestion воркфлоу отработал успешно (все ноды зеленые).

3. Зайдите в Pinecone Console и проверьте, что в индексе есть записи (Vectors > 0).

4. Dimensions индекса должны быть 1536 (для text-embedding-3-small).

Telegram "chat not found" ошибка ▼

1. Убедитесь, что вы написали боту /start перед тестом.

2. Проверьте, что chat_id правильно передается между нодами (откройте Output каждой ноды в Executions).

3. Если используете Group Chat — боту нужны права на чтение сообщений.

Сколько стоит использование бота? ▼

Pinecone: Бесплатный тариф (до 100K векторов).

OpenAI Embeddings: ~$0.02 за 1 млн токенов (очень дешево).

OpenRouter (DeepSeek): ~$0.14 за 1 млн токенов ввода. В среднем $0.01 за 100 запросов.

n8n Cloud: От $20/месяц. Self-hosted — бесплатно.

Итого для небольшой компании: $2-10/месяц.

Можно ли использовать свои PDF/документы вместо JSON? ▼

Да! Для этого нужно модифицировать Ingestion воркфлоу:

1. Замените "Load Legal Data" на ноду Read Binary File или Google Drive.

2. Добавьте ноду Extract from PDF для парсинга PDF.

3. Остальная логика (chunking, embeddings, upsert) останется без изменений.
