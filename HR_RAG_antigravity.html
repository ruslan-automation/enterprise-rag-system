Создание бота и получение токена

⏱️ ~5 мин

Первым делом нам нужно зарегистрировать самого бота в Telegram.

Инструкция:

  1. Откройте Telegram и найдите бота @BotFather.
  2. Нажмите Start и отправьте команду /newbot.
  3. Придумайте имя (например: HR Legal Assistant) и юзернейм (например: hr_legal_rag_bot).
  4. BotFather пришлёт вам длинный API Token. Скопируйте его — он понадобится в n8n.

  2. Нажмите кнопку Add Credential.
  3. В поиске напишите Telegram API и выберите его.
  4. Вставьте скопированный токен в поле Access Token и сохраните.

Эмбеддинги

Cohere — превращение текста в векторы

⏱️ ~5 мин

Для RAG-бота нужен сервис эмбеддингов — он превращает текст в числовые векторы для поиска по смыслу. Мы используем Cohere — бесплатный и отлично работает с русским языком!

💡
OpenRouter (LLM) подключим отдельно — в шаге 4. Сейчас нам нужен только Cohere для эмбеддингов.

Cohere (Эмбеддинги) — для поиска по базе Free!

Cohere используется только для эмбеддингов — превращения текста в векторы для поиска. Модель embed-multilingual-v3.0 отлично работает с русским языком и полностью бесплатна на Trial тарифе (до 1000 запросов/мин).

  1. Перейдите на dashboard.cohere.com/api-keys.
  2. Зарегистрируйтесь (можно через Google).
  3. На странице API Keys уже будет сгенерирован Trial Key.
  4. Скопируйте ключ (начинается с ... — нажмите "Show" чтобы увидеть).
💡
Почему Cohere, а не OpenAI?
1. Бесплатно — Trial ключ без ограничений по времени
2. Мультиязычная модель — embed-multilingual-v3.0 лучше работает с русским текстом
3. Не нужна оплата — не надо привязывать карту
📥 Куда вставлять в n8n:
  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите Cohere API.
  3. Вставьте скопированный ключ в поле API Key.
  4. Назовите его Cohere и нажмите Save.
⚠️
Важно для Pinecone! Модель embed-multilingual-v3.0 создаёт векторы размерностью 1024 (не 1536 как у OpenAI). При создании индекса Pinecone укажите Dimensions = 1024.

Pinecone

Настройка векторной базы данных

⏱️ ~5 мин

Pinecone — это облачная векторная база данных. Она хранит ваши документы в виде чисел (векторов) и позволяет мгновенно находить самые релевантные фрагменты по смыслу запроса.

Создание индекса: Free Tier!

  1. Перейдите на app.pinecone.io и зарегистрируйтесь (можно через Google).
  2. Нажмите Create Index.
  3. Заполните поля:
    • Name: legal-assistant
    • Dimensions: 1024 (стандарт для Cohere embed-multilingual-v3.0)
    • Metric: cosine
  4. Нажмите Create Index и дождитесь создания.
  5. Скопируйте API Key из раздела API Keys в левом меню.
  6. Скопируйте Host URL вашего индекса (выглядит как https://legal-assistant-xxxxx.svc.xxxxx.pinecone.io).
⚠️
Запишите оба значения! API Key и Host URL понадобятся в HTTP Request нодах воркфлоу. Host URL у каждого индекса уникальный.
📥 Куда вставлять в n8n:
  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите Header Auth.
  3. В поле Name введите: Api-Key
  4. В поле Value вставьте ваш Pinecone API Key.
  5. Назовите Credential: Pinecone API Key и сохраните.

Host URL вашего индекса понадобится в HTTP Request нодах — вставьте его вместо YOUR-INDEX.svc.YOUR-ENV.pinecone.io в URL-поле.

💡
Бесплатный тариф Pinecone покрывает до 100,000 векторов — этого хватит на сотни документов!

LLM

Подключение OpenRouter (Мозг бота)

⏱️ ~10 мин

OpenRouter — это шлюз к лучшим нейросетям (DeepSeek, GPT-4o, Claude). Он совместим с OpenAI API, поэтому подключается через стандартный OpenAI Credential в n8n, но с кастомным URL.

💡
Рекомендуемая модель: deepseek/deepseek-chat — быстрая, точная и очень дешёвая ($0.14/1M токенов). Для RAG идеально!
Альтернатива: google/gemini-2.0-flash-001 — с бесплатным лимитом для старта.

Шаг 1: Регистрация и получение ключа

  1. Зайдите на openrouter.ai и зарегистрируйтесь.
  2. Перейдите в Credits → Add credits и пополните баланс ($2+).
  3. Перейдите в Keys → Create Key.
  4. Скопируйте ключ — он начинается с sk-or-...
Регистрация на OpenRouter
Создание API Key
Ключ создан — скопируйте его

Шаг 2: Подключение в n8n

OpenRouter использует тот же формат API, что и OpenAI. Поэтому в n8n мы создаём OpenAI Credential, но меняем Base URL:

📥 Настройка Credential в n8n:
  1. В n8n перейдите в меню слева → Credentials.
  2. Нажмите Add Credential.
  3. В поиске напишите OpenAI API (да, именно OpenAI — НЕ "OpenRouter"!).
  4. Вставьте ваш ключ sk-or-... в поле API Key.
  5. Раскройте секцию Options (кнопка внизу формы).
  6. В поле Base URL впишите:
    https://openrouter.ai/api/v1
  7. Назовите Credential: OpenRouter.
  8. Нажмите Save.
⚠️
Самая частая ошибка! Если забыть указать Base URL — n8n будет отправлять запросы на api.openai.com вместо OpenRouter, и ваш ключ sk-or-... не сработает. Обязательно проверьте: Options → Base URL = https://openrouter.ai/api/v1
📋 Где будет использоваться:

Этот Credential нужен в ноде Legal AI Response в Chat воркфлоу. Он отвечает за генерацию ответов пользователю на основе найденного контекста.

Оплата

Оплата OpenRouter через Zarub 🇷🇺

⏱️ ~10 мин

Если у вас российская карта — OpenRouter напрямую не примет оплату. Используйте сервис Zarub — он выпускает виртуальную зарубежную карту для оплаты AI-сервисов.

💡
Если у вас уже есть зарубежная карта (или крипта) — пропустите этот шаг и оплатите OpenRouter напрямую через Credits.

Шаг 1: Регистрация в Zarub

  1. Перейдите в официальный бот Zarub в Telegram.
  2. Нажмите Start и следуйте инструкциям.
Бот Zarub в Telegram — нажмите "Перейти в кошелёк"

Шаг 2: Пополнение и выпуск карты

  1. Пополните баланс — минимум 10$. Можно с российской карты или криптой.
  2. В приложении нажмите "Выпустить карту".
  3. Нажмите Зелёную кнопку (перевести на карту) — чтобы деньги попали на счёт карты.
  4. Нажмите Фиолетовую кнопку — она покажет реквизиты вашей виртуальной карты (номер, срок, CVV).
Зелёная кнопка — перевод, Фиолетовая — реквизиты карты

Шаг 3: Оплата OpenRouter

  1. Откройте openrouter.ai/credits.
  2. Нажмите Add credits.
  3. Введите реквизиты карты Zarub (номер, срок, CVV).
  4. Пополните на $2-5 для старта — этого хватит на тысячи запросов DeepSeek.
⚠️
Важно: Карта Zarub подходит также для оплаты OpenAI, Anthropic, Google Cloud и других AI-сервисов.

7
Google

Подключение Google Sheets

⏱️ ~15 мин

Google Sheets используется как источник данных для RAG-бота. Вы загружаете регламенты в таблицу, а n8n автоматически читает их и отправляет в Pinecone.

⚠️
Это самый сложный этап! Нужно создать OAuth-приложение в Google Cloud Console. Рекомендуем следовать видео-инструкции ниже.
ВАЖНО: Смотреть разбор настройки Google API (таймкод 04:30)
7.1

Создание проекта в Google Cloud Console

  1. Перейдите на console.cloud.google.com.
  2. Нажмите Select a project → New Project.
  3. Дайте имя (например: n8n-rag-bot) и нажмите Create.
7.2

Включение Google Sheets API

  1. В левом меню выберите APIs & Services → Library.
  2. Найдите Google Sheets API и нажмите Enable.
7.3

Настройка OAuth Consent Screen

  1. Перейдите в APIs & Services → OAuth consent screen.
  2. Выберите External и нажмите Create.
  3. Заполните:
    • App name: n8n RAG Bot
    • User support email: ваш email
    • Developer contact email: ваш email
  4. Нажмите Save and Continue до конца.
  5. ВАЖНО! На вкладке Test Users добавьте свой email (Add Users).
⚠️
Самая частая ошибка! Если не добавить свой email в Test Users — авторизация в n8n не сработает. Обязательно добавьте себя!
7.4

Создание OAuth Credentials

  1. Перейдите в APIs & Services → Credentials.
  2. Нажмите Create Credentials → OAuth Client ID.
  3. Тип: Web application.
  4. Скопируйте Client ID и Client Secret.
7.5

Подключение в n8n

📥 Куда вставлять в n8n:
  1. В n8n перейдите в Credentials → Add Credential.
  2. Найдите Google Sheets OAuth2 API.
  3. Вставьте Client ID и Client Secret.
  4. Скопируйте OAuth Redirect URL из n8n (показывается в форме Credential).
  5. Вернитесь в Google Cloud Console → ваш OAuth Client → Authorized redirect URIs → вставьте URL из n8n.
  6. Нажмите Sign in with Google в n8n и авторизуйтесь.
💡
Совет: Если вы используете n8n Desktop (localhost), Google может потребовать перейти по ссылке вручную. Просто скопируйте URL из адресной строки и вставьте в поле n8n.

Данные

Подготовка и загрузка данных

⏱️ ~5 мин

Для демо мы используем встроенные данные в воркфлоу загрузки. Вам НЕ НУЖНО ничего готовить — 7 статей законов (УК РФ, ТК РФ, ГК РФ) уже прописаны.

💡
Как это работает:
Данные → разбиваются на чанки (фрагменты) → превращаются в векторы (эмбеддинги через Cohere) → загружаются в Pinecone. При запросе пользователя бот ищет ближайшие по смыслу чанки и передаёт их LLM для генерации ответа.

Вариант A: Hardcoded данные (Демо)

В воркфлоу Ingestion уже зашиты 7 статей. Просто запустите его.

Вариант B: Google Sheets (Продакшн)

  1. Создайте Google Таблицу с колонками: id, content, type, code, article.
  2. Заполните строки вашими регламентами / статьями закона.
  3. В Ingestion воркфлоу замените узел "Load Legal Data" на ноду Google Sheets.
  4. Выберите вашу таблицу и лист → данные подтянутся автоматически.

Тестирование

⏱️ ~5 мин

Финальный этап — проверяем, что всё работает!

  1. Убедитесь, что Chat воркфлоу Active (зелёный переключатель).
  2. Откройте вашего бота в Telegram.
  3. Напишите /start, затем любой вопрос.
  4. Попробуйте примеры:
    • "Какое наказание за кражу?"
    • "Как расторгнуть договор аренды?"
    • "Могут ли уволить за прогул?"
  5. Бот должен ответить со ссылками на конкретные статьи законов!
🔴
Бот не отвечает? Проверьте:
  • Workflow активирован (зелёный переключатель Active)
  • Все Credentials настроены правильно
  • Нет ошибок во вкладке Executions
  • URL Pinecone и API ключ подставлены верно
  • OpenRouter Credential имеет Base URL
🎉
Поздравляем! Если бот отвечает — вы только что запустили свою RAG-систему! Теперь можно загрузить реальные документы и использовать бота для HR-консультаций.

❓ Частые вопросы и проблемы

Бот не реагирует на сообщения ▼

1. Убедитесь, что Chat воркфлоу активирован (зелёный переключатель).

2. Проверьте вкладку Executions — там видны все ошибки.

3. Откройте Telegram Trigger ноду и проверьте Credential.

4. Деактивируйте и снова активируйте воркфлоу.

Ошибка "Invalid API Key" в OpenRouter ▼

1. Проверьте, что ключ начинается с sk-or-.

2. Убедитесь, что на балансе OpenRouter есть средства ($1+).

3. Проверьте Base URL: https://openrouter.ai/api/v1

4. Попробуйте создать новый ключ.

Pinecone: "Index not found" или пустые результаты ▼

1. Проверьте URL индекса в HTTP Request нодах.

2. Убедитесь, что Ingestion воркфлоу прошёл успешно (все ноды зелёные).

3. В Pinecone Console проверьте Vectors > 0.

4. Dimensions индекса = 1024 (для Cohere embed-multilingual-v3.0).

Telegram "chat not found" ▼

1. Убедитесь, что написали боту /start перед тестом.

2. Проверьте, что chat_id передаётся между нодами (Output каждой ноды в Executions).

3. Если Group Chat — боту нужны права на чтение.

Сколько стоит использование? ▼

Pinecone: Бесплатно (до 100K векторов).

Cohere Embeddings: Бесплатно (Trial Key).

OpenRouter (DeepSeek): ~$0.14/1M токенов ввода. ~$0.01 за 100 запросов.

n8n Cloud: От $20/мес. Self-hosted — бесплатно.

Итого: $2-10/месяц для небольшой компании.

Можно ли загрузить свои PDF/документы? ▼

Да! Замените "Load Legal Data" на ноду Read Binary File или Google Drive.

Добавьте Extract from PDF для парсинга.

Остальная логика (chunking, embeddings, upsert) останется без изменений.
